# Skill: Codex Rules

When generating code follow these rules:

1. Prefer server components in Next.js
2. Avoid unnecessary client components
3. Create reusable components
4. Separate UI from business logic
5. Use TypeScript types everywhere
6. Keep files under 300 lines when possible
7. Prefer clean and readable code over clever code

When implementing new features:

1. create database schema
2. add migration
3. create Supabase queries
4. build UI
5. connect UI to database
